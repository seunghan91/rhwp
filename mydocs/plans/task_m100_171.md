## 2. 현재 상태

`src/serializer/hwpx/header.rs`의 `write_header()`는 한컴2020 레퍼런스 템플릿(`templates/empty_header.xml`)을 그대로 반환한다.  
즉, `Document.doc_info`의 글꼴/글자모양/문단모양/스타일 정보가 직렬화 결과에 전혀 반영되지 않는다.

현재 `section.xml`의 `<hp:p paraPrIDRef="0" styleIDRef="0">` 및 `<hp:run charPrIDRef="0">`도 고정값(ID=0)이므로,  
header.xml이 동적 생성되어야 ID 연동이 의미를 갖는다.

## 3. 작업 목표

`Document.doc_info` IR을 OWPML(`<hh:*>` 네임스페이스) 속성으로 매핑하여 `header.xml`을 동적 생성한다.

| IR 필드 | OWPML 요소 |
|---------|------------|
| `font_faces` (Vec<Vec<Font>>) | `<hh:fontfaces>` / `<hh:fontface>` / `<hh:font>` (언어별 7개) |
| `border_fills` | `<hh:borderFills>` |
| `char_shapes` | `<hh:charProperties>` / `<hh:charPr>` |
| `tab_defs` | `<hh:tabProperties>` |
| `numberings` / `bullets` | `<hh:numberings>` |
| `para_shapes` | `<hh:paraProperties>` / `<hh:paraPr>` |
| `styles` | `<hh:styles>` / `<hh:style>` |

`section.xml`의 paraPrIDRef / charPrIDRef / styleIDRef도 각 문단·글자 모양의 ID로 동적 연동한다.

## 4. 작업 범위

### 포함
- `src/serializer/hwpx/header.rs`: `write_header()` 전면 교체 (IR → XML 동적 생성)
- `src/serializer/hwpx/section.rs`: paraPrIDRef / charPrIDRef / styleIDRef 동적 연동
- 단위 테스트: 각 IR 필드별 XML 구조 검증
- 라운드트립 검증: 글꼴/스타일 보존 확인

### 제외
- HWP(바이너리) 직렬화 변경 없음
- 컨트롤(표/이미지/그림) 직렬화 (#172)
- CI 검증 파이프라인 (#173)

## 5. 검증 계획

| 방법 | 내용 |
|------|------|
| 단위 테스트 | ZIP 추출 후 fontfaces/charPr/paraPr XML 요소 존재 및 속성값 검증 |
| 라운드트립 | `cargo run --example hwpx_roundtrip` — 글꼴명/크기/스타일 ID 보존 확인 |
| 한컴 오픈 | `tools/verify_hwpx.py` — 글꼴 정상 적용 시각 확인 |
