# HWPX section.xml 완전 동적화 (secPr + 다중 run + Control dispatcher)

## 이슈
[#186](https://github.com/edwardkim/rhwp/issues/186) — HWPX section.xml 완전 동적화

## 작업 브랜치
`local/task186`

## 배경

#171(header.xml 동적화) 완료 후 section.xml 쪽이 여전히 template 기반.  
#182 코드가 로컬에 없으므로 해당 내용(표/그림 직렬화)을 포함하여 본 작업에서 함께 구현한다.

## 현재 상태

- `section.rs`: template(`empty_section0.xml`) + string replacement 방식
  - 첫 문단의 secPr은 템플릿 고정값 사용 (pagePr width/height/margin 하드코딩)
  - 단일 run (`charPrIDRef` 하나) — char_shapes 구간 분할 없음
  - Control::Table / Control::Picture 처리 없음 (`\u{0002}` 무시)
- `content.rs`: BinData 지원 구조(`BinDataEntry`) 있으나 실제 이미지 미연결
- 표/그림 serializer 파일 없음

## 목표

| 범위 | 내용 |
|------|------|
| A — secPr 동적화 | `SectionDef.page_def` → `<hp:pagePr>`, `<hp:footNotePr>`, `<hp:endNotePr>`, `<hp:pageBorderFill>` |
| B — 다중 run | `char_shapes[]` UTF-16 offset 기준으로 `<hp:run charPrIDRef>` 구간 분할 |
| C — 표 직렬화 | `Control::Table` → `<hp:tbl>` + 셀/행 직렬화 |
| D — 그림 직렬화 | `Control::Picture` → `<hp:pic>` + BinData ZIP 삽입 + content.hpf 연동 |

## 완료 기준

- 기존 serializer::hwpx 31개 테스트 회귀 없음
- `char_shapes.len() > 1` 샘플에서 run 개수 == char_shapes 개수
- 표가 포함된 HWPX 라운드트립: `<hp:tbl>` 정상 출력
- 그림이 포함된 HWPX 라운드트립: `<hp:pic>` + BinData ZIP 정상 출력
- form-002.hwpx 라운드트립 시 한컴에서 표 내용 정상 표시
